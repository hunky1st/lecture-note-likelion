# 3교시(href속성 target속성(attribute=속성=특성))

- anchor태그는 <a></a>로 표기
- 앵커란? 하이퍼링크!
- 각각의 파일에서 메인페이지로 이동을 마지막에 넣는 과정.
  <a href="/index.html">메인 페이지></a>

- 하이퍼링크는 바디 안에 들어가야 함. 바디 밖에 넣어도 html은 작동하는 이유는 html이 자동으로 작동하기 때문임. 따라서 개발자도구와 내가 짠 코드를 항시 비교해봐야함.
- 새 창(속성) 속성 = 어트리뷰트임
- target="\_blank" 속성으로 새창으로 열림이 됨. 언더바를 빼면 blank라는 이름의 새 창을 열어주게 됨. 결과적으로 새 창인것처럼 보이는 것은 동일하지만.. 언더바는 확실히 새창열기이고 언더바가 없으면 블랭크라는 이름의 창을 만들어 열어줌. 다음에 같은 창을 열면 기존에 블랭크라는 이름의 할당된 창을 열어주는 이상한 일이 생기게 됨.
- mdn페이지를 상실 잘 확인하길. 공식 레퍼런스 이므로 슬비쌤의 노션보다도 업데이트가 빠르고 당연히 모든 내용이 있음. **기승전 mdn**
- 특정 태그만 지우는 법 커맨드 + d
- 특정 태그만 블럭잡고 커맨드 쉬프트 l -> 해당 문자열의 모든 문자열이 다 선택됨.
- `https://`, `mailto:`, `tel:` 모두 다 하이퍼링크임
- 전역속성
- **"동일출처 url"** 기술면접시 자주 물어보는 내용.
- 값을 지정하지않는 불리언 어트리뷰트를 지정할 수 있음. ex download라고만 적음. 예를들어 pdf를 뷰어로 열리게 하거나, 다운로드 받는 방법을 구별 할 수 있음.
- mdn 언제나 열어서 보기
- 북마크 채널에 스닙펫(https://snippet-generator.app/?description=&tabtrigger=&snippet=&mode=vscode) 제너레이터 자주사용하는 속성을 글자 몇개만 치면 나오게끔 할당할 수 있음.
  이후 cmd+shift+p로 snippet설정을 열어서 제너레이터에서 생성된 코드를 산입하면 할당완료.

# 6교시

- 06정의형 목록
- 시맨팅마크업은 무엇인가? = 인간도 이해할 수 있고 기계도 이해할 수 있는 의미에 맞는 마크업을 하는 행위.
- 예를 들어 <div>이형민</div> -> 그냥 이형민인데 무슨 의미인지 모름.그런데 <p>이형민</p>을 하면 이형민이 뭔지는 모르지만 단락 이형민이 됨.
- <dl>요소는 dt와 dd요소를 자식으로 가질 수 있음
- 인간이 코드를 구별하기 쉬운 가독성도 중요함. 가령 <div>태그만 사용하면 제작은 용이하나 인간이 코드를 읽는 가독성이 떨어짐.
- <div>태그는 순서와 뜻이 없는 컨테이너 같은 개념의 태그 비정의형, non시멘틱임. 언제나 정의형 시멘틱이 정답이라는것은 아니지만. 고민은 필요
- <dt>제목 <dd>설명 개념 -> 디자인이 쉬워지도록 dt와 dd를 묶는다.
- 11 섹션과 메인 요소(6항목 이후 잠시 11항목으로 점프)
- <header> <footer> <main> <aside> 등등의 요소 등장
- <a>는 기능이 있지만 <ul>,<ol>,<li>,<dl>,<dt>,<dd>는 기능이 있다기 보단 시멘틱(의미)를 가지는 태그임.
- <header> 머릿말 - 웹 페이지 상단에 큰 틀을 잡는 메뉴, 검색창 등이 나열되는 경우 등
- <footer> 바닥말 - 사이트를 마무리 하는 부분. 예를들어 카피라이트, 연락처, sns링크 등이 있는
- <main> html문서에서 단 한번만 사용할 수 있음. 주요 컨텐츠. 메인 안에 다시 헤더와 푸터가 들어갈 수 있음.
- <aside> 부가 정보에 해당. 예를들어 사이드바 라던가.
- <section> 1장 2장 등등의 구분, <article> 완결된, 독립된 정보를 명시할 때 사용. 예를들어 블로그에서 포스트를 올려서 완결된 정보를 제공하는 경우.
- <section><article>양자의 구분은 너무 빠져들지 않기를 바람. 그렇게까지 중요하지 않음. 심지어 둘 다 안써도 무관함.
- <hr /> br태그와 유사한 성격. hr은 호리존탈 룰러 = 수평선임.
- WAI-ARIA는 랜드마크 규칙.
- contentinfo
